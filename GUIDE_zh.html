Photoshop → Unity UI 工具 v2.4.2

將 Photoshop 排版好的遊戲 UI 轉成 Unity uGUI + TextMeshPro Prefab。文字圖層保留為可編輯的 TMP 節點，非文字圖層逐一輸出為 PNG Sprite。

安裝 主流程 PS 匯出對話框 圖層命名規則 Layout Group 自動排版 Unity 匯入工具 專案資料夾結構 文字與材質 版本更新 常見問題

安裝

PS　Photoshop 端

JSX 腳本不需要安裝，兩種執行方式擇一：

方式 A（推薦）：放入 PS Scripts 資料夾

將 PhotoshopExporter/ 資料夾內的 JSX 複製到下列路徑，重啟 Photoshop 後即可從選單叫出：

Windows：C:\Program Files\Adobe\Adobe Photoshop [版本]\Presets\Scripts\
Mac：    /Applications/Adobe Photoshop [版本]/Presets/Scripts/

之後從 File > Scripts > PhotoshopUiPackageExporter 執行。

方式 B：每次手動瀏覽

在 Photoshop 執行 File > Scripts > Browse…，選擇 PhotoshopExporter/PhotoshopUiPackageExporter.jsx。

Unity　Unity 端

將整個 Assets/Editor/PhotoshopUiImporter/ 資料夾複製到 Unity 專案的 Assets/Editor/ 底下，Unity 會自動編譯。

完成後從選單開啟：

Tools > Photoshop UI Importer > Importer_v2
前置需求：Unity 專案需安裝 TextMeshPro（Package Manager 搜尋 TextMeshPro 安裝即可）。

主流程

1
整理 PSD

確認圖層為英文命名。如需批次改名，執行 PhotoshopLayerAutoNamer.jsx。

2
PS 匯出

執行 PhotoshopUiPackageExporter.jsx，指定 PNG 資料夾與 Layout JSON 路徑，點 Export。

3
Unity 套用 Package

在 Importer_v2 選擇 Package 資料夾，點「套用 Package」，工具自動找 JSON 與圖片。

4
建立專案資料夾

填寫「專案資料夾名稱」，點「建立專案資料夾」，自動建立子目錄結構並生成 SpriteAtlas。

5
指定字型與材質

至少指定「預設 TMP Font Asset」。有現成字框材質庫時，可填入「TMP 材質球資料夾」。

6
Generate Prefab

點「Validate」確認資料，再點「Generate Prefab」完成輸出。

圖層命名規則

Anchor / Pivot 標記

在圖層名稱中加入標記（不分大小寫），Unity 匯入時自動套用對應的 RectTransform Anchor 與 Pivot：

標記Anchor 效果
ANCHOR_TL / ANCHOR_TOP_LEFT左上角固定
ANCHOR_TR / ANCHOR_TOP_RIGHT右上角固定
ANCHOR_BL / ANCHOR_BOTTOM_LEFT左下角固定
ANCHOR_BR / ANCHOR_BOTTOM_RIGHT右下角固定
ANCHOR_C / ANCHOR_CENTER中心固定
ANCHOR_ML / ANCHOR_LEFT左中固定
ANCHOR_MR / ANCHOR_RIGHT右中固定
ANCHOR_TC / ANCHOR_TOP上中固定
ANCHOR_BC / ANCHOR_BOTTOM下中固定
STRETCH_X / STRETCH_HORIZONTAL水平延展
STRETCH_Y / STRETCH_VERTICAL垂直延展
STRETCH_FULL / STRETCH_BOTH全方向延展

未加任何標記時，工具根據元素在父容器中的相對位置自動推斷 Anchor（靠左 / 靠右 / 置中 / 延展等）。

Pivot 可另行指定（可與 Anchor 標記組合）：

標記效果
PIVOT_TL左上角 pivot
PIVOT_C / PIVOT_CENTER中心 pivot
PIVOT_BL左下角 pivot（其他方向以此類推）

特殊前綴與標記

前綴 / 標記作用
IGNORE_ 或 [ignore]匯出時完全跳過此圖層。
REF_參考用圖層，匯出時跳過。
IMG_圖片前綴（前綴本身會從 PNG 檔名中移除）。
TXT_文字前綴（前綴本身會從節點名稱中移除）。

文字圖層輸出控制

個別圖層可在名稱中指定輸出方式，覆蓋對話框的預設設定：

名稱中含效果
[PNG] [IMAGE] [IMG] TXTIMG_強制輸出為 PNG 圖片
[TMP] [TEXT] TMP_ TXT_強制保留為 TMP 節點

材質球標記（文字圖層）

在文字圖層名稱加上材質標記，指定 Unity 要套用的 TMP 材質球：

TXT_title_[MAT:title_yellow_outline]
TXT_score_[MAT:reward_white_red_outline]

Unity 端 Skin Map 可將 materialToken 對應到不同的 TMP Material Preset。若圖層有 Layer Style 的 Stroke 效果，工具會自動產生 materialToken，無需手動標記。

假厚度標記（文字圖層）

在文字圖層名稱加入 [THICK:N]，N 為向下偏移的像素數，Unity 會自動疊兩層 TMP 模擬立體感：

TXT_title_[THICK:3]
TXT_score_[THICK:2]

上層為原始文字，下層為顏色加深（× 0.55）的陰影層，偏移量對應 PS 設計稿像素。同時支援 X 軸偏移：[THICK:2:1]（格式：[THICK:offsetY:offsetX]）。

v2.4 起 PS 端 JSX 已自動解析 [THICK:N] 與 [THICK:N:M]，名稱中加標記即可，不需手動編輯 JSON。

Layout Group 自動排版 v2.4

圖層群組可自動轉為 Unity 的 HorizontalLayoutGroup 或 VerticalLayoutGroup，並自動套上 ContentSizeFitter。

v2.4.2 起停用自動去重：先前依「相同寬高」判斷重複圖會誤判（譬如同寬度但不同字的文字、同尺寸但不同 icon），故 v2.4.2 起每個圖層各自輸出獨立 PNG。若需要圖片共用，請在 PS 將圖層名稱統一。

偵測規則

支援兩種方式，命名標記優先於自動偵測：

方式條件結果
命名標記群組名稱含 [H] 或 [HLAYOUT]（不分大小寫）強制水平 layout
命名標記群組名稱含 [V] 或 [VLAYOUT]（不分大小寫）強制垂直 layout
自動偵測群組有 2 個以上子層，且所有子層 Y 位置一致（±3px 容差）水平 layout
自動偵測群組有 2 個以上子層，且所有子層 X 位置一致（±3px 容差）垂直 layout
無不符合上述任一條件普通群組，不加 Layout Component

實際 PSD 圖層命名範例

情境 A：標記強制（不依賴位置）

Settings_Page/
├── BtnBar_[H]/              ← 強制水平
│   ├── Btn_Home.png
│   ├── Btn_Settings.png
│   └── Btn_Shop.png
└── Menu_List_[V]/           ← 強制垂直
    ├── Menu_Play.png
    ├── Menu_Help.png
    └── Menu_Quit.png

情境 B：自動偵測（不寫標記）

Toolbar/                     ← 子層 Y 全相同 → 自動水平
├── Icon_1.png  (x=100, y=50)
├── Icon_2.png  (x=150, y=50)
└── Icon_3.png  (x=200, y=50)

情境 C：結合其他標記（多標記共存）

TopBar_[H][ANCHOR_TC]/       ← 水平 layout + 上方置中錨點
├── Coin.png
├── Diamond.png
└── Ticket.png

Spacing / Padding 自動計算

  • Spacing：依子層排序後，計算相鄰子層的像素間距，取中位數作為 spacing。
  • Padding：群組邊界與最外層子層邊界的距離，分別套用到 Left / Right / Top / Bottom。
  • 所有值由 PS 設計稿座標自動算出，不需手動輸入。

重複圖去重（v2.4.2 已停用）

v2.4.0 ~ v2.4.1 曾以「相同 width × height」為判斷依據自動共用 imagePath。但實務上此判斷不準確：

  • 同寬度但不同字的文字節點被誤判為同一張圖。
  • 共用後也會干擾 Layout Spacing 中位數計算，造成排版錯誤。

v2.4.2 起取消自動去重，每個圖層各自輸出一張 PNG。若多個圖層真的指向同一張圖，請將圖層名稱命名一致（如 btn_bg + 副本），工具會以 btn_bg_002、btn_bg_003 的方式分別匯出，仍可由設計師後續手動合併或在 Unity 端用相同 Sprite。

專案資料夾結構

點「建立專案資料夾」後，工具在 Assets/Temp/{專案名稱}/ 下自動建立以下結構，並同步建立 SpriteAtlas asset：

Assets/Temp/{專案名稱}/
├── Animation/
├── Atlas/
│   └── SpriteAtlas/
│       ├── SpriteAtlas.spriteatlas   ← 自動建立，打包來源為 SpriteAtlas/ 資料夾
│       ├── Base/                      ← 美術圖（預設語言）
│       ├── CHS/                       ← 簡體中文文字圖
│       ├── CHT/                       ← 繁體中文文字圖
│       └── EN/                        ← 英文文字圖
├── Font/
├── Fx/
│   ├── Material/
│   └── Texture/
├── Prefab/
├── Spine/
└── TimeLine/
SpriteAtlas 自動設定：建立後 SpriteAtlas 的 Objects for Packing 已指向 SpriteAtlas/ 資料夾，PS 匯出的圖片放入對應語言子資料夾後，在 Unity 手動點「Pack Preview」即可打包。

Unity 匯入工具（Importer_v2）

開啟路徑：Tools > Photoshop UI Importer > Importer_v2

1. Photoshop UI Package

選擇 Photoshop 匯出的 Package 根資料夾，點「套用 Package」。工具自動掃描找出 layout JSON（優先找 layout.json，其次找 *_layout.json）以及 PNG 圖片資料夾。按「只驗證資料」可在不生成 Prefab 的情況下確認資料正確。

2. Unity 生成位置

填寫「專案資料夾名稱」（例如 Setting_Page），有兩個按鈕可用：

  • 「建立專案資料夾」（新專案建議用）：自動建立完整子目錄結構、生成 SpriteAtlas，並填入輸出路徑。詳見 專案資料夾結構。
  • 「套用標準輸出路徑」（既有資料夾用）：只填入 Atlas 與 Prefab 路徑，不建立資料夾。

圖片來源若已在 Unity 專案 Assets 底下，工具直接使用來源路徑，不重複複製。「參考解析度」預設自動跟隨 Layout JSON 中的 Canvas 尺寸。

3. 文字、材質與換皮

欄位用途
預設 TMP Font Asset所有文字節點的 fallback 字型。UI 含文字時必填。
預設 TMP 材質球文字的 fallback 材質。Stroke 效果自動衍生的材質也以此為基底，建議填寫。
TMP 材質球資料夾（選填）指定一個存放現成字框材質球的資料夾。Generate 時優先從此資料夾比對，找到相近的直接重用，找不到才自動新建。設定後視窗會即時顯示找到幾顆。
Skin Map（選填）將特定 Sprite 名稱替換為換皮版本。
換皮工具：同一區塊下還有「換皮工具（美術圖覆蓋）」，可指定美術來源資料夾與 Unity 目標資料夾，執行「換皮掃描」後自動覆蓋同名 PNG。找不到對應美術的檔名會列出成待辦清單。

4. 執行

  • Validate：讀取 Layout JSON，驗證節點數量與資料格式。
  • 預覽（Dry-run）：顯示節點數、是否含文字、圖片資料夾狀態與預計輸出路徑，不執行任何操作。
  • Generate Prefab：匯入圖片、生成 Prefab，完成後在 Project 視窗自動選取並 ping 出 Prefab。

文字與材質球

TMP 文字節點自動流程

  1. PS 文字圖層的 Layer Style → Stroke 效果，匯出 JSON 時寫出 outlineColor、outlineWidth 與自動 materialToken。
  2. PS 文字圖層的 Layer Style → Color Overlay 效果，填色顏色寫入 color 欄位，Unity 端套用至 TMP 頂點色。
  3. Unity Generate 時，依序執行材質球解析：
    1. 若設定了「TMP 材質球資料夾」，先掃描資料夾內所有材質球，找到色差 < 8% 且寬度誤差 < 15% 的即直接重用，不新增。
    2. 未找到匹配 → 從預設材質球複製一份，套用 _OutlineColor / _OutlineWidth，儲存至 Assets/Temp/{名稱}/Font/GeneratedMaterials/。
    3. 無 Stroke 資料的文字節點直接套用預設材質球，不做任何處理。
材質球重用邏輯：同一頁面若多個文字用了接近的字框顏色，工具會自動共用同一顆材質球，避免 GeneratedMaterials 資料夾膨脹。

手動指定材質球

PS 圖層名稱：TXT_title_[MAT:title_yellow_outline]
            TXT_score_[MAT:reward_white_red_outline]

在圖層名稱加上 [MAT:名稱] 標記，Unity 端會優先以此 token 為鍵，從材質球資料夾或 GeneratedMaterials 中尋找對應材質。

版本更新

PS　PS 端更新

執行 PhotoshopUiPackageExporter.jsx，對話框右上角點 Check for Updates。工具會連線 GitHub 比對版本號，確認後自動下載最新 JSX 並覆蓋當前檔案，下次執行即為新版。

  • Windows：使用 PowerShell WebClient 下載。
  • Mac：使用 curl 下載。

Unity　Unity 端更新

在 Importer_v2 視窗標題列點 從 GitHub 更新工具。工具連線 GitHub Contents API 取得最新腳本清單，顯示版本對比後，確認即自動下載所有 .cs 並覆蓋本地，Unity 自動重新編譯。

更新後腳本內容會立即覆蓋，Unity 編譯完成前工具暫時無法使用，屬正常現象。

常見問題

匯出後 PNG 有些圖沒有輸出？

可能原因：圖層為 Clipping Mask、Adjustment Layer（色調、曲線等）、或 Ignore/REF 前綴圖層，這些均會被跳過。執行後的 alert 會顯示 skipped 數量。

同一個圖層名稱重複出現，PNG 有 _002 後綴？

工具會自動去重命名，重複出現的名稱加上三位數後綴（layer_002、layer_003…），這是正常行為。建議在 PSD 中保持圖層名稱唯一。

Generate Prefab 提示「請先指定預設 TMP Font Asset」？

Layout 中含文字節點（type = text）時，至少要在 Unity 端指定一個 TMP Font Asset 作為 fallback，工具才能繼續。

匯入後圖片位置跑偏？

確認「參考解析度」與 Canvas 尺寸一致，工具預設自動讀取 layout.json 中的 Canvas 設定，一般無需手動調整。若有自訂 Canvas Scaler，需確認 Match Width or Height 設定與 PS 設計稿一致。

圖片來源已在 Assets 內，不想重複複製？

直接選擇 Assets 內的資料夾作為 Package 資料夾，工具偵測到圖片已在專案內時，會直接使用原路徑，不重複複製。

PS 端更新後腳本空白或損壞？

可能是網路中斷或 PowerShell 執行被防毒軟體阻擋。請至 GitHub 手動下載最新 JSX：
https://github.com/Wayne188-yiching/PS_To_Unity_v2

Unity 端更新顯示「無法解析 GitHub 回應」？

請確認電腦可連線至 api.github.com。部分企業網路會封鎖 API 呼叫，可手動從 GitHub 下載 zip 並覆蓋 Assets/Editor/PhotoshopUiImporter/。

點「建立專案資料夾」後 SpriteAtlas 沒出現？

確認已填寫「專案資料夾名稱」再按按鈕。建立後若 Project 視窗未更新，執行 Assets → Refresh（或按 Ctrl+R）即可看到 SpriteAtlas.spriteatlas。

Generate 後文字顏色不正確？

工具讀取 PS Color Overlay 效果的顏色作為 TMP 頂點色。若文字使用的是 Gradient Overlay（漸層），目前版本僅支援純色，漸層上下色需等待後續版本支援。

材質球資料夾設定後，Generate 還是一直新建材質球？

比對條件為：RGB 每個色板差異 < 8%，且寬度誤差 < 15%。若庫中材質球顏色差異超出此範圍則視為不符，工具會自動新建。可確認庫中材質球的 _OutlineColor 是否與 PS 設定的 Stroke 色相近。

文字在 PS 顯示 36pt，Unity 卻變成 29pt？v2.4.1 已修正

v2.4.0 之前使用 textItem.size，這個 API 拿到的是「設計時原始字體」，不含 Free Transform 縮放。若文字圖層被自由變形縮放過，PS 字元面板顯示的是「縮放後的有效字體」（impliedFontSize），兩者會不同。
v2.4.1 起改用 ActionDescriptor 讀取 impliedFontSize，與 PS 字元面板顯示完全一致。

明明設定 TMP，文字卻變成圖片？v2.4

v2.4 起新增「Auto-export non-Source-Han fonts as PNG」自動分流。若文字圖層使用非思源系列字型（思源黑體/宋體、源泉圓體、Noto CJK 之外的字型），會自動輸出為 PNG，避免 Unity 缺字型造成顯示異常。
若要強制以 TMP 輸出，可在對話框取消勾選此選項，或在圖層名稱加入 [TMP] / TXT_ 等手動標記。

Layout Group 沒有自動偵測成功？v2.4

自動偵測需要群組內所有子層的 X（垂直 layout）或 Y（水平 layout）座標一致（±3px 容差）。若子層位置略有偏移超出容差，可在群組名稱加 [H] 或 [V] 強制指定方向。命名標記永遠優先於自動偵測。
